# RULPredict

Remaining-useful-life forecasting from health-index degradation signals. Uses a Ridge lag model to forecast future health values and estimates when the signal crosses a maintenance threshold.

## Run it

```bash
pip install -r requirements.txt
python train.py
streamlit run app.py
```

Synthetic turbofan health data is generated by default. To use real telemetry data, upload a CSV in the sidebar or place it in `data/raw/` and wire it through `load_real()`.

## How the forecasting works

- Ridge regression on lagged sensor readings (lags 1, 2, 3, 7, 14, 28)
- No calendar features — RUL data is cycle-indexed, not timestamped, so day-of-week and day-of-year add noise without signal
- Recursive forecasting: the model predicts one step ahead, appends the prediction to the input, and repeats
- Threshold crossing detection returns the first cycle where forecast health falls below a configurable maintenance threshold
- The RUL estimate is 0 if the last observed value is already below threshold

## Inside the app

The app shows the observed and forecast health trajectory, estimated RUL, risk band (low/medium/high/critical), recent degradation slope, and evaluation metrics on a chronological holdout set.

## Structure

```
src/
  core.py          RidgeRegression, Standardizer, metrics
  data.py          synthetic health-index generator
  model.py         lag feature pipeline and recursive forecasting
  evaluate.py      metric persistence
  persist.py       model save/load
train.py           training entry point
app.py             Streamlit dashboard
```

## Caveats

The Ridge model is a baseline. Production RUL systems typically use temporal CNNs, LSTMs, or survival models that handle censored data natively. The recursive forecasting approach also accumulates prediction errors — multi-step uncertainty propagation would better be handled by a dedicated sequence model.
